## Shadows

Shadows are the main way to give the illusion of depth [^1]. Based on the concept of physical surfaces and shadows, simulating how light interacts with objects in the real world. Elements with higher elevation cast larger or more pronounced shadows. Shadows are generally _soft_ and are used to make element stand-out from background content.

<Shadows />

| Value    | Name             | Usage                             | Token             |
| -------- | ---------------- | --------------------------------- | ----------------- |
| shadow-s | Low elevation    | Used for buttons, links and forms | `var(--shadow-s)` |
| shadow-m | Medium elevation | Used for cards, headers and lists | `var(--shadow-m)` |
| shadow-l | High elevation   | Used for floating items           | `var(--shadow-l)` |

## Blurs

<Blur />

For visual depth, interface components can have a inner background blur applied [^2]. Usually this used and reserved for fixed navigational elements such as 'pill' or 'filter' navigations and buttons.

| Value  | Name        | Usage                             | Token           |
| ------ | ----------- | --------------------------------- | --------------- |
| blur-s | Low fade    | Used for buttons, links and forms | `var(--blur-s)` |
| blur-m | Medium fade | Used for cards, headers and lists | `var(--blur-m)` |
| blur-l | High fade   | Used for floating items           | `var(--blur-l)` |

## Border radius

The borders are used for _squarish_ elements. Rounded edge feel a little softer and more approachable and help guide the views eyes more smoothly across the page aiding scannability increasing the overall natural flow.
